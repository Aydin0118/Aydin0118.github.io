# Important

OpenClaw 是一款本地优先、高权限、可自主执行系统操作的开源 AI 智能体，核心优势是能直接替你完成电脑操作

但也正因如此，**隐私与安全风险极高**

- 为维护网络安全与利益，**请保持高度警惕**


## Openclaw本质：



### Orchestrator（大脑层）

- 理解意图、拆解任务、规划执行步骤
- Invoke large models (GPT/Claude/local models) for execution
- 维护持久记忆（本地向量库），上下推理

### Gateway（协议桥）

- input bin ：HTTP、WebSocket、RPC、IM 机器人适配器
- 做权限校验、指令路由、日志记录，连接大脑与执行端
- Pipeline


### Pi-embedded（执行端 / 四肢）

- 在本地环境实际执行操作：读写文件、运行命令、操控浏览器、发邮件、调用 API
- 拥有系统级权限，可访问你的所有文件、进程、网络资源


## OpenClaw 的风险来自高权限 + 自主执行 + 外部输入 + 开源生态的叠加
### 1. 提示词注入 / 指令越狱（最常见）
   原理：攻击者在网页、邮件、PDF、文档中嵌入恶意指令，OpenClaw 读取后被误导，忽略安全规则，执行越权操作
#### 风险：
> 窃取 API 密钥、密码、浏览器记录、钱包信息

> 批量删除文件、格式化磁盘、执行 rm -rf 等高危命令
> 用你的身份发邮件、发消息、转账

### 2. 恶意工具 / 技能投毒
原理：ClawHub 技能市场任何人可上传插件，恶意代码伪装成 “文件整理”“邮件助手” 等工具
> 风险：安装后自动执行后门、窃取数据、挖矿、控制你的电脑
### 3. 敏感信息泄露（配置与存储）
原理：
API 密钥、数据库密码、服务凭证常以明文存放在 .env、config.json 中

记忆库（向量数据库）存储大量个人数据，未加密则易被窃取
> 风险：密钥被盗刷、第三方服务被劫持、个人隐私数据泄露。

### 4. 模型幻觉导致误操作（非攻击但致命）
原理：AI 理解错误、过度自信、上下文丢失，执行非预期高危操作。
> 风险：误删重要文件、错误执行数据库脚本、批量发送敏感内容。

### 5. 权限过大与信任边界模糊
原理：默认常给完整系统权限，无精细控制；公网暴露、多设备共享时易被接管。
> 风险：设备完全受控、数据被窃取、被用于网络攻击。

## 隐私安全使用要点
### 1. 权限最小化（核心）
关闭完整系统权限，只开放必要能力：
- 仅允许访问特定目录（如 ~/Documents/OpenClaw），禁止访问根目录、系统目录、浏览器配置目录
- 禁用 shell、sudo、rm、format 等高危命令
- 禁止自动执行，所有操作必须人工确认
- 在 Web UI → TOS 标签页，关闭不必要的权限开关
### 2. 严格隔离与沙箱

- 用 Docker / 虚拟机 运行 OpenClaw，与宿主系统完全隔离。
- 用 Linux 命名空间、Capabilities 限制权限，仅分配必要能力。
- 不要在生产服务器、公司设备、多人共享设备上运行。
### 3. 输入与工具安全

- 禁止 OpenClaw 自动读取陌生网页、邮件附件、未知文档
> 安装技能前：
> 只从官方 / 可信来源下载，审查代码（开源可查）
> 不安装需要完整系统权限、shell 执行、网络外发的工具
> 所有外部输入先人工审核，再让 OpenClaw 处理

### 4. 敏感信息保护
- API 密钥、密码绝不明文存储：使用环境变量、密钥管理工具（如 1Password）、加密配置文件
- 记忆库、日志加密存储，定期清理过期数据
- **个人信息（身份证、手机号、邮箱）在输入前脱敏**

### 5. 版本与更新
只使用官方稳定版，不使用测试版、修改版

及时更新，关注官方安全公告与漏洞修复
